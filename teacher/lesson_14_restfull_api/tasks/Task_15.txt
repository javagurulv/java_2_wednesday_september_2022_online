Тема для самостоятельного изучения.

Отсылка HTTP REST запросов из Java с помощью
класса RestTemplate из библиотеки Spring.

https://www.baeldung.com/rest-template

Идея заключается в следующем:

- у приложения есть REST API с помощью которого можно выполнить
  все команды (то же самое что можно сделать через UI);
- нужно создать клиентский код, который позволит посылать
  REST вызовы в приложение;
- поднять приложение;
- написать похожие на существующие acceptance тесты, которые
  вместо выполнения запросов через сервисы посылают REST запросы
  к приложению, получают ответы и проверяют их. Таким оьразом
  проводя тестирование.

Зачем это нужно? Во первых если ваш REST API публичный, то вам
нужно его протестировать.

Во вторых, если ваш REST API дублирует функциональность UI,
то тестировать функциональность приложения через REST API
намного быстрее и надёжнее чем через UI. REST API имеет дело
только с данными и поэтому он намного быстрее чем WEB UI.
